# hpc_mcc — HPC scripts for BANKSY parameter sweep

Scripts and outputs for running the BANKSY tissue domain parameter sweep on the
University of Kentucky Morgan Computing Cluster (MCC).

---

## Files

### Scripts

| File | Purpose |
|------|---------|
| `run_multipleBanksy.sh` | SLURM batch script — loads HPC modules and submits `banksy_parameter_sweep.R` |
| `banksy_parameter_sweep.R` | Runs the full 4 × 4 lambda × resolution sweep; writes the Seurat object and summary CSVs |
| `banksy_sweep_visualize.R` | Reads `banksy_sweep.qs2` and generates one diagnostic PDF per parameter combination |

### Outputs (generated on HPC, synced locally)

| File / Directory | Generated by | Contents |
|-----------------|--------------|----------|
| `markers_all_combinations.csv` | `banksy_parameter_sweep.R` | Marker genes for every cluster across all 16 combinations (columns: `lambda`, `resolution`, `cluster`, `gene`, standard Seurat marker stats) |
| `sweep_cluster_assignments.csv` | `banksy_parameter_sweep.R` | One row per cell; 16 columns (`lambda{x}_resolution{y}`) with the cluster label assigned under each combination |
| `plots/banksy_lam{##}_res{##}.pdf` | `banksy_sweep_visualize.R` | 16 single-page PDFs, one per combination (see layout below) |

---

## Workflow

### Step 1 — Submit the parameter sweep

```bash
sbatch run_multipleBanksy.sh
```

The SLURM job requests **16 CPUs / 192 GB RAM / 24 h** on the `normal` partition
(account `coa_lajo247_uksr`). It loads the following modules before running R:

| Module | Required by |
|--------|------------|
| `r-4.4.0-gcc-9.3.0-7nl5q44` | R itself |
| `curl-7.76.1-gcc-9.3.0-37a6bcy` | R `curl` package |
| `gmp-6.2.1-gcc-9.3.0-iqbexj7` | `leidenAlg` (via GMP headers) |
| `imagemagick-7.0.8-7-gcc-9.3.0-zygjkan` | R `magick` package |
| CMake (added to `PATH`) | `fs` and other compiled dependencies |

The script calls `banksy_parameter_sweep.R` with no arguments; it uses the
default input and output paths (see below).

### Step 2 — Run the parameter sweep (`banksy_parameter_sweep.R`)

**Input** (default, override with `--input`):
```
20260320_fullobj_correctedChloe_sct_SPLIT.qs2   # SCT_SPLIT-normalised Seurat object
```

**Parameter grid:**
- Lambdas: `0.2, 0.4, 0.6, 0.8`
- Resolutions: `0.1, 0.3, 0.5, 1.0`
- **16 combinations total**

**Per lambda:** `RunBanksy` (k_geom = 15, use_agf = TRUE, split.scale = TRUE,
group = "Slide") → `RunPCA` (50 PCs) → `RunUMAP` (30 dims, seed 42) →
`FindNeighbors`. Each UMAP is stored under a lambda-specific reduction key
(e.g. `umap_banksy_lam02` for λ = 0.2).

**Per resolution:** `FindClusters` (Leiden, BANKSY SNN graph) →
`FindAllMarkers` (Wilcoxon, SCT_SPLIT assay). Cluster labels are written back
to the base object as metadata columns named `lambda{x}_resolution{y}`
(e.g. `lambda0.2_resolution0.1`).

**Outputs** (default directory `spatial/banksy_sweep/`, override with `--out_dir`):

```
spatial/banksy_sweep/
  banksy_sweep.qs2               # Seurat object with all 16 metadata columns + 4 UMAP reductions
  markers_all_combinations.csv   # stacked marker table (~16 × n_clusters × n_genes rows)
  sweep_cluster_assignments.csv  # cell × combination cluster assignment matrix
```

To run manually:
```bash
Rscript banksy_parameter_sweep.R \
  --input  /path/to/object.qs2 \
  --out_dir spatial/banksy_sweep
```

### Step 3 — Generate diagnostic PDFs (`banksy_sweep_visualize.R`)

Reads `banksy_sweep.qs2` and writes one PDF per combination into
`spatial/banksy_sweep/plots/`.

**PDF layout (18 × 16 in):**

```
┌─────────────────┬────────────────────┬─────────────────────┐  ← 1/3 height
│   UMAP          │  Domain composition│ Treatment composition│
│  (no legend)    │  (cell types)      │  (Adu / IgG)        │
└─────────────────┴────────────────────┴─────────────────────┘
│          Spatial domains per brain (6 panels, nrow = 2)     │  ← 2/3 height
└─────────────────────────────────────────────────────────────┘
```

- UMAP uses the lambda-specific reduction (`umap_banksy_lam{##}`)
- Spatial plots use the Polychrome `kelly` palette (up to 22 clusters);
  `createPalette()` is used as fallback for higher-resolution runs with > 22 clusters
- Treatment colours: Aducanumab `#E07B39`, IgG `#7B68AE`

**Output naming:** `banksy_lam{##}_res{##}.pdf`
(e.g. `banksy_lam02_res01.pdf` for λ = 0.2, resolution = 0.1;
`banksy_lam08_res10.pdf` for λ = 0.8, resolution = 1.0)

```bash
Rscript banksy_sweep_visualize.R --out_dir spatial/banksy_sweep
```

## UMAP reduction keys

| Lambda | Reduction stored in Seurat object |
|--------|----------------------------------|
| 0.2    | `umap_banksy_lam02` |
| 0.4    | `umap_banksy_lam04` |
| 0.6    | `umap_banksy_lam06` |
| 0.8    | `umap_banksy_lam08` |

## Metadata column naming

Cluster labels in `banksy_sweep.qs2` follow the pattern
`lambda{lam}_resolution{res}` using one decimal place, e.g.:

```
lambda0.2_resolution0.1
lambda0.2_resolution0.3
...
lambda0.8_resolution1.0
```
